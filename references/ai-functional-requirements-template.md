# AI-facing Functional Requirements Template

This template is for Codex/developer continuation. It must be precise enough for another Codex session to continue implementation without the original chat.

## Document rules

- Do not include schedules, dates, sprint plans, staffing plans, or budget estimates.
- Preserve the dual-version strategy: Game Studio/Phaser master first, WeChat Mini Game adaptation second.
- Keep requirement IDs stable.
- Mark unknowns as `Open question` instead of guessing.
