# Reader-friendly Functional Requirements Template

This template is for the project owner. It should be easy to read, review, and correct.

## Document rules

- Use plain Chinese.
- Explain what the player experiences.
- Avoid implementation jargon unless the owner must make a product decision.
- Do not include project schedules, dates, or development cycles.
- Keep the same feature set as the AI-facing document.

## Recommended structure
